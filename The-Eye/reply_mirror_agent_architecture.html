Structured features
Amount z-score, time-of-day, velocity, balance delta, payment method encoding
Contextual features
GPS-txn distance, NLP sentiment/urgency, social graph centrality, recipient novelty
↓ feature vectors distributed to all agents simultaneously
3 — Intelligent agent layer
Transaction anomaly
Isolation Forest · per-user rolling baselines · amount & frequency outliers
Behavioral profiling
User habit model · merchant novelty · time-of-day drift scoring
Geospatial
GPS vs. txn location · impossible travel · jurisdiction shifts
NLP & comms
Phishing signals · urgency cues · social engineering patterns in SMS/email
Graph & network
Transaction graph · ring detection · money mule path analysis
Temporal patterns
Time-series habits · sudden schedule shifts · burst detection
↓ each agent emits a fraud probability score + confidence
4 — Meta-orchestrator
Orchestrator agent
Weighted voting across all agent scores · asymmetric cost calibration (FP ≠ FN penalty) · dynamic threshold per challenge level · final fraud / legitimate decision
↓ decisions logged  ·  ↺ feedback loop updates agents between levels
5 — Adaptation & memory
Drift detection engine
ADWIN / DDM monitors concept drift · triggers per-agent retraining when patterns shift · cross-level performance tracking
Memory bank
Fraud pattern embeddings per level · few-shot generalization to unseen attack vectors · inter-level pattern delta analysis
↑ memory & updated models fed back into agents and orchestrator before each new level
↓ cost-aware threshold optimization
6 — Output layer
Output generator
Precision–recall threshold sweep · asymmetric cost minimization · fraudulent Transaction ID list → submission file
